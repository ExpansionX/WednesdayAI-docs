[vLLM](https://docs.vllm.ai) is a high-throughput inference server for open-weight models. OpenClaw connects to vLLM via its OpenAI-compatible API.

## Prerequisites

- A running vLLM server with its OpenAI-compatible API enabled
- The server must be reachable from the OpenClaw gateway host

## Configuration

```yaml
providers:
  vllm:
    enabled: true
    baseUrl: "http://localhost:8000/v1"
    model: "meta-llama/Llama-3.1-8B-Instruct"
    apiKey: ""
    timeout: 120000
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `enabled` | boolean | `false` | Enable this provider |
| `baseUrl` | string | — | Base URL of the vLLM OpenAI-compatible endpoint |
| `model` | string | — | Model ID as loaded by your vLLM server |
| `apiKey` | string | `""` | API key if vLLM is configured with `--api-key` |
| `timeout` | integer ms | `120000` | Request timeout — increase for large models |

## Running vLLM

```bash
docker run --runtime nvidia --gpus all \
  -v ~/.cache/huggingface:/root/.cache/huggingface \
  -p 8000:8000 \
  vllm/vllm-openai:latest \
  --model meta-llama/Llama-3.1-8B-Instruct
```

## Verify the connection

```bash
openclaw models status --provider vllm --probe
```

<Note>
vLLM does not expose a model list endpoint by default. The `--probe` flag sends a test completion to verify connectivity.
</Note>
