GitHub Copilot can be used as an AI provider through the OpenClaw gateway, routing requests through your organisation's Copilot subscription.

## Prerequisites

- A GitHub account with an active GitHub Copilot subscription (Individual, Business, or Enterprise)
- The `openclaw` CLI version 1.3.0 or later

## Authentication

Use the interactive device-flow login:

```bash
openclaw models auth login-github-copilot
```

This opens a browser for GitHub OAuth. On completion, the token is stored in your config automatically.

Alternatively, set the token manually:

```yaml
providers:
  copilot:
    token: "ghu_..."
```

<Warning>
GitHub Copilot tokens expire. Run `openclaw models auth login-github-copilot` again if you see authentication errors.
</Warning>

## Configuration

```yaml
providers:
  copilot:
    enabled: true
    token: ""           # or GITHUB_COPILOT_TOKEN env var
    endpoint: "https://api.githubcopilot.com"
    organization: ""   # optional: restrict to an org subscription
```

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `enabled` | boolean | `false` | Enable this provider |
| `token` | string | — | GitHub Copilot auth token (or `GITHUB_COPILOT_TOKEN`) |
| `endpoint` | string | `"https://api.githubcopilot.com"` | API endpoint |
| `organization` | string | — | Restrict to an org's Copilot subscription |

## Available models

Models vary by subscription tier:

| Model ID | Notes |
|----------|-------|
| `gpt-4o` | Default for Business/Enterprise |
| `gpt-4o-mini` | Faster, lower cost |
| `claude-sonnet-4-6` | Available on Enterprise |
| `o3-mini` | Reasoning model |

Run `openclaw models status --provider copilot` to see the current list for your subscription.

## Verify the connection

```bash
openclaw models status --provider copilot --probe
```
