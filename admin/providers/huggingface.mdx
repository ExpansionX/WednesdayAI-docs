# HuggingFace

WednesdayAI supports the HuggingFace Inference API (serverless endpoints), giving access to thousands of public and private models hosted on the Hub.

## Prerequisites

Create a **fine-grained token** at [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) with the **"Make calls to Inference Providers"** permission. Set it on the gateway host using either accepted variable name:

```bash
# Either variable is accepted:
echo 'HUGGINGFACE_HUB_TOKEN=hf_...' >> ~/.openclaw/.env
# or
echo 'HF_TOKEN=hf_...' >> ~/.openclaw/.env
```

<Note>
  Both `HUGGINGFACE_HUB_TOKEN` and `HF_TOKEN` are accepted. If both are set, `HUGGINGFACE_HUB_TOKEN` takes precedence.
</Note>

## Configuration

```json5
{
  models: {
    providers: {
      huggingface: {
        apiKey: "HUGGINGFACE_HUB_TOKEN", // or "HF_TOKEN"
        models: [
          // add model IDs from the HuggingFace Hub
          "meta-llama/Meta-Llama-3.1-8B-Instruct",
          "mistralai/Mistral-7B-Instruct-v0.3",
        ],
      },
    },
  },
}
```

<Warning>
  HuggingFace serverless inference is only available for models that support it. Gated models (e.g. Llama) require you to accept the model's terms on the Hub before your token can access them.
</Warning>

## Selecting a model

```json5
{
  agents: {
    defaults: {
      model: { primary: "huggingface/meta-llama/Meta-Llama-3.1-8B-Instruct" },
    },
  },
}
```

HuggingFace model IDs follow the `owner/model-name` format used on the Hub.

<Tip>
  Model IDs support routing suffixes: append `:fastest`, `:cheapest`, or `:<provider>` to control which inference provider handles the request. For example, `meta-llama/Llama-3.3-70B-Instruct:fastest` routes to whichever provider currently has the lowest latency.
</Tip>

## Inference router endpoint

Instead of targeting a specific backend, you can use HuggingFace's inference router, which automatically selects the cheapest or fastest available provider:

```json5
{
  models: {
    providers: {
      huggingface: {
        apiKey: "HUGGINGFACE_HUB_TOKEN",
        baseUrl: "https://router.huggingface.co/v1",
        models: [
          "meta-llama/Llama-3.3-70B-Instruct",
        ],
      },
    },
  },
}
```

<Note>
  The router endpoint honours `:fastest` and `:cheapest` suffixes on model IDs. Omitting a suffix lets the router apply its default selection policy.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="401 Unauthorized">
    The token is invalid or expired. Generate a new one at [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) and update `~/.openclaw/.env`.
  </Accordion>
  <Accordion title="403 Forbidden — gated model">
    The model requires accepting terms on the HuggingFace Hub before your token can access it. Visit the model page and accept the access agreement.
  </Accordion>
  <Accordion title="Model does not support inference API">
    Not all Hub models support serverless inference. Check the model page for an "Inference API" badge, or deploy the model to a dedicated endpoint.
  </Accordion>
</AccordionGroup>
