# Venice AI

Venice AI is a privacy-focused inference provider. Prompts and responses are not logged or used for training, making it suitable for sensitive workloads.

## Prerequisites

Create an API key at [venice.ai](https://venice.ai) and set it on the gateway host:

```bash
echo 'VENICE_API_KEY=vapi_xxxxxxxxxxxx' >> ~/.openclaw/.env
```

<Note>
  Venice API keys follow the format `vapi_xxxxxxxxxxxx`. Generate yours at [venice.ai](https://venice.ai).
</Note>

## Configuration

```json5
{
  models: {
    providers: {
      venice: {
        apiKey: "VENICE_API_KEY", // references the env var
        models: [
          "llama-3.3-70b",
          "llama-3.1-405b",
          "mistral-31-24b",
          "qwen-2.5-vl-72b",
          "deepseek-r1-671b",
        ],
      },
    },
  },
}
```

<Note>
  `apiKey` is the name of the environment variable, not the key value itself. Keep secrets in `.env`, not in committed config files.
</Note>

## Selecting a model

```json5
{
  agents: {
    defaults: {
      model: { primary: "venice/llama-3.3-70b" },
    },
  },
}
```

| Model | Privacy tier | Notes |
| ----- | ------------ | ----- |
| `llama-3.3-70b` | Fully private | Llama 3.3 70B — strong general-purpose |
| `llama-3.1-405b` | Fully private | Llama 3.1 405B — highest-capacity open model |
| `mistral-31-24b` | Fully private | Mistral 3.1 24B — efficient mid-size model |
| `qwen-2.5-vl-72b` | Fully private | Qwen 2.5 VL 72B — vision-language model |
| `deepseek-r1-671b` | Fully private | DeepSeek R1 671B — extended reasoning |

<Note>
  Venice offers two privacy tiers. **Fully private** — no data retention, no logging, no training use. **Anonymised proxy** — metadata stripped but requests may pass through shared infrastructure. The `venice` provider uses the fully private tier by default. The full model catalog (including anonymised-tier models) is listed at [venice.ai](https://venice.ai).
</Note>

<Tip>
  Venice does not log prompts or responses on the fully private tier. For compliance-sensitive deployments, verify Venice's current data handling policy at [venice.ai/privacy](https://venice.ai/privacy).
</Tip>

Run `openclaw models list --provider venice` for the full current catalog.

## Troubleshooting

<AccordionGroup>
  <Accordion title="401 Unauthorized">
    The API key is invalid or revoked. Generate a new one at [venice.ai](https://venice.ai) and update `~/.openclaw/.env`.
  </Accordion>
  <Accordion title="Model not found">
    Venice's model catalog may differ from other providers. Check the Venice dashboard for currently available models.
  </Accordion>
</AccordionGroup>
